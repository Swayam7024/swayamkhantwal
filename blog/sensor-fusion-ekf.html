Sensor Fusion with Extended Kalman Filter

The Problem: Unreliable Sensors

Building autonomous systems requires knowing where you are with accuracy. GPS can tell you rough position, but it drifts. IMU sensors measure acceleration and rotation, but they accumulate errors over time. Barometers measure altitude but are affected by weather.

No single sensor is reliable enough for autonomous navigation. The solution? Sensor Fusion—combining multiple sensors intelligently to produce better estimates than any individual sensor.

Real-world impact: Using sensor fusion, I improved position estimation accuracy from ±5 meters (GPS alone) to ±0.5 meters (fused estimates). That's a 10x improvement!

Why Extended Kalman Filter?

The Extended Kalman Filter (EKF) is the industry standard for sensor fusion in autonomous systems because it:

  • Predicts state: Uses a motion model to predict where the drone will be
  • Corrects with measurements: Updates predictions based on sensor readings
  • Estimates uncertainty: Tracks how confident it is about each estimate
  • Handles nonlinearity: Works with dynamics that aren't strictly linear (like drone kinematics)
  • Real-time capable: Computationally efficient enough for embedded systems

Understanding the EKF Algorithm

The Two Stages

The EKF operates in two alternating stages:

1. Prediction (Time Update)

Between sensor measurements, the filter predicts the drone's next state using a motion model:

  • If accelerometer reads +2 m/s², predict velocity will increase by 2 m/s
  • If velocity is 10 m/s, predict position will move 10 meters in that direction
  • Also account for uncertainty: "I'm pretty sure about this, but not 100%"

2. Correction (Measurement Update)

When a new sensor measurement arrives, the filter:

  • Compares the measurement with what was predicted
  • Calculates the difference (innovation or residual)
  • Updates the state estimate based on how trustworthy the sensor is
  • Updates the uncertainty estimate

Practical Implementation

State Vector

I defined the state as: [x, y, z, vx, vy, vz, pitch, roll]

This captures position (x,y,z), velocity, and orientation angles needed for autonomous flight.

Sensor Models

GPS Measurement

GPS gives us position directly: z_gps = [x_meas, y_meas, z_meas] but with noise of about ±5m

IMU Measurement

Accelerometer and gyroscope provide motion information:

  • Accelerometer: a = dv/dt (very accurate short-term)
  • Gyroscope: ω = dθ/dt (drifts over time)

Barometer Measurement

Altimeter provides altitude: z_alt = barometer_reading with ±0.5m accuracy

Covariance Tuning

The key to a good EKF is properly tuning the covariance matrices:

  • Process Noise (Q): How much we trust the motion model (low Q = high trust)
  • Measurement Noise (R): How much we trust sensors (low R = high trust)

For my system:

  • GPS: R = very large (signals weak, trust less)
  • IMU: R = small (very accurate, trust more)
  • Barometer: R = medium (decent accuracy for altitude only)

Real Performance Results

Without Sensor Fusion (GPS Only)

Horizontal Position Error: ±5 to 10 meters
Vertical Error: ±2 to 5 meters
Problem: Can't follow a straight flight path, drifts in random directions

With EKF Fusion

Horizontal Position Error: ±0.3 to 0.7 meters
Vertical Error: ±0.2 to 0.4 meters
Benefit: Can reliably follow GPS waypoints and hover in place

Challenges and Solutions

Challenge 1: GPS Dropouts

Problem: When flying under trees or in urban canyons, GPS signal fails.

Solution: The EKF naturally handles this! Without GPS measurements, it relies on IMU predictions. The state uncertainty grows, but the system remains stable for several minutes using only IMU.

Challenge 2: Biased Accelerometer Readings

Problem: Mounting vibrations and temperature changes bias accelerometers.

Solution: Implemented bias estimation as a separate state in the filter. The EKF learns and corrects for these biases automatically.

Challenge 3: Computational Cost

Problem: Full EKF with 12+ states runs at 10 Hz on Raspberry Pi. We needed 50 Hz for smooth control.

Solution: Optimized matrix operations, reduced state vector to essential states only, and implemented an unscented Kalman filter (UKF) approximation for speed.

Key Takeaways

  • Sensor fusion is a necessity: No single sensor is good enough for autonomous systems
  • EKF is practical: Despite being mathematical, it's surprisingly practical to implement
  • Tuning matters: Getting the noise covariances right is critical for performance
  • Uncertainty quantification is valuable: Knowing not just where you are, but how confident you are
  • Real-world deployment is different: Always test with actual sensor data in deployment environment
